1.Introduction

The Software Design Specification (SDS) document will specify a high-level view of the architecture of our system, and on the interaction between the user and the system and detailing a low-level view of each component of the software and how the components interact with each other. This document's purpose is to provide a high-level design framework around which to build our project FITMENT. It also provides a list of requirements against which to test the final project and determine whether we were able to successfully implement the system according to design.

3.Description of Design Components

The following bulleted outline provides a basic overview of the purpose and architecture of our system's major components. The tables in the remainder of section 3 will provide more details on each component:

• Database: We will be using relational database. It will contain tables to represent various project data.

• Class Library: The Class Library will be a package of classes common to the Server Software and the Client Software. The server software will use these class definitions to know how to store the objects in the database. The client software will use the class definitions in order to ask for and display the correct information about each class type

• User

• Admin

• Register

• Buyer

• Item

• Ordered item

• Review


• Reporting Application: A web application which will use the information in the database to produce useful reports for users.

• Server Software: Our server software will directly manipulate the contents of the database, based on commands from the Server Procedure Proxy.

• Server Procedure Proxy: The Server Procedure Proxy will provide an interface for the clients to request information or submit updates to the server.

• Client Software: The Client Software will reside on either a desktop computer or an android. Its purpose is to present data to the user as requested, and provide an interface so that the user can easily update project information. It will handle conflict resolution automatically when possible, and otherwise allow the user to choose which version of an item to keep.


• Android Client: The Android client will allow the user to view basic information about and make simple changes to data regarding projects assigned to them.

• Desktop Client: With significantly more computing power, screen size, and storage space than the android, the desktop client
